The proposed system is an AI-based retinal diagnostic pipeline that analyzes fundus images to detect eye diseases. It uses a two-stage hierarchical classification approach, where a lightweight deep learning model first filters normal cases and then performs detailed disease classification on abnormal images.
Designed for fast and efficient screening
Reduces computational load using staged processing

 Stage 1–3: Input, Preprocessing & Feature Extraction
The system takes a retinal fundus image as input from a camera or dataset.
The image is preprocessed by resizing, normalizing pixel values, and applying augmentation (rotation, flip) to improve generalization.
It is then passed through MobileNetV2 (pretrained on ImageNet), where deep features such as blood vessels, lesions, and optic disc structures are extracted.
The backbone layers are frozen, and only the classifier is trained for efficiency.




Stage 4–5: Hierarchical Classification & Output
The extracted features are fed into a two-stage classification pipeline.
The gatekeeper model (binary classification) first determines whether the retina is normal or abnormal.
If abnormal, the image is passed to the specialist model (multiclass classification), which identifies diseases like diabetic retinopathy, glaucoma.
The system outputs the predicted disease label along with a confidence score, enabling accurate and efficient diagnosis suitable for edge deployment.

The various tools used are:

The system is developed using PyTorch for building and training deep learning models, along with Torchvision for accessing pretrained MobileNetV2 and handling datasets and transformations.





Various algorithms used can be listed as:
1. Transfer Learning (MobileNetV2)
The system uses MobileNetV2 pretrained on ImageNet to leverage learned visual features instead of training from scratch. The feature extraction layers are frozen (requires_grad = False), allowing only the final classification layers to be trained. This reduces training time, requires less data, and improves generalization, especially for medical images where datasets are limited.

2. Convolutional Neural Network (CNN) – Feature Extraction
MobileNetV2 acts as the core CNN architecture that extracts high-level features from retinal images. It uses depthwise separable convolutions, which reduce computational cost while maintaining accuracy. The network captures important retinal patterns such as blood vessels, lesions, and structural abnormalities.

3. Binary Classification Algorithm (Gatekeeper Model)
A binary classification approach is used in the first stage to classify images as normal or abnormal.
Output layer: Single neuron
Activation: Sigmoid
Loss Function: BCEWithLogitsLoss
This stage acts as a filter, ensuring only abnormal images are passed to the next stage, reducing unnecessary computation.


4. Multiclass Classification Algorithm (Specialist Model)
The second stage performs multiclass classification to identify specific retinal diseases.
Output layer: Number of disease classes
Activation: Softmax (via CrossEntropyLoss)
Loss Function: CrossEntropyLoss with class weights
Class weighting is applied to handle dataset imbalance and improve performance on underrepresented diseases.
 5. Data Augmentation Techniques
To improve model robustness, various augmentation techniques are applied during training:
Random rotation
Horizontal and vertical flipping
Color jitter (brightness/contrast changes)
These techniques help the model generalize better to real-world variations in retinal images.
